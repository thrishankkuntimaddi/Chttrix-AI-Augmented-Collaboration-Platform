PROJECT OVERVIEW: ChttrixCollab - AI-Augmented Collaboration Platform
═══════════════════════════════════════════════════════════════════════════════

TECHNOLOGY STACK:
─────────────────
Frontend:  React 18, TailwindCSS, Socket.io-client, Axios
Backend:   Node.js, Express.js, Socket.io, MongoDB/Mongoose
Auth:      JWT (Access + Refresh tokens), OAuth
Real-time: Socket.io for live messaging and notifications

KEY FEATURES:
─────────────
✓ Multi-workspace collaboration
✓ Real-time messaging (DM & Channels)
✓ Task management with tracking
✓ Notes and knowledge base
✓ Company updates/blog system
✓ AI-powered chat assistance (ChttrixAI)
✓ 🔒 Role-based access control (Owner/Admin/Member)
✓ 📧 Advanced invite system (email + shareable links)
✓ 🚪 Invite lifecycle management (pending/accepted/revoked)
✓ 👥 Workspace member management (add/remove)
✓ 🗑️ Workspace deletion with cascading cleanup
✓ Email invitations and domain verification
✓ Audit logs and activity history
✓ Message threading and reactions
✓ File attachments and media sharing

ARCHITECTURE PATTERNS:
──────────────────────
• MVC pattern on backend (Models, Controllers, Routes)
• Component-based architecture on frontend
• Context API for global state management
• Service layer for API abstraction
• Custom hooks for reusable logic
• Socket.io for real-time bidirectional communication
• JWT-based stateless authentication
• Workspace-scoped routing (all features under /workspace/:id/...)
• Multi-tenant channel isolation (channels belong to single workspace)
• Admin-only operations with role validation (backend + frontend)

ROUTING STRUCTURE:
─────────────────
✓ All workspace features are scoped under /workspace/:workspaceId/...
✓ Routes:
  • /workspace/:workspaceId/home              - Dashboard
  • /workspace/:workspaceId/channels          - Channel list
  • /workspace/:workspaceId/channel/:id       - Specific channel
  • /workspace/:workspaceId/messages          - Direct messages
  • /workspace/:workspaceId/dm/:id            - Specific DM
  • /workspace/:workspaceId/tasks             - Task management
  • /workspace/:workspaceId/notes             - Notes
  • /workspace/:workspaceId/notes/:id         - Specific note
  • /workspace/:workspaceId/updates           - Company updates

✓ Public routes (outside workspace):
  • /login, /register-company                 - Authentication
  • /workspaces                               - Workspace selector
  • /verify-email, /reset-password            - Account management
  • /accept-invite, /join-workspace           - Workspace joining
  • /admin/company                            - Company admin panel

WORKSPACE-CHANNEL ISOLATION:
────────────────────────────
✓ Every channel belongs to exactly ONE workspace
✓ Channels are queried with workspace filter: Channel.find({ workspace: workspaceId })
✓ Default channels (#general, #announcements) are created per workspace
✓ No cross-workspace data leakage
✓ Channel members are subset of workspace members

PERMISSIONS & ROLES:
───────────────────
✓ Workspace roles: owner, admin, member
✓ Owner: Full control (only one per workspace)
✓ Admin: Can invite, manage members, create channels
✓ Member: Can view, participate, limited management
✓ Backend validation on all admin-only operations
✓ Frontend UI guards for role-specific features

INVITE SYSTEM ARCHITECTURE:
───────────────────────────
✓ Email-based invites (multiple recipients, role selection)
✓ Shareable link invites (one-click join)
✓ Invite status tracking: pending → accepted/revoked
✓ 7-day expiration with validation
✓ SHA-256 token hashing for security
✓ One-time use enforcement
✓ Admin-only revocation capability
✓ Pending invites list in modal
✓ Auto-join default channels on acceptance

WORKSPACE DELETION:
──────────────────
✓ Owner-only operation
✓ Workspace name confirmation required
✓ Cascading deletions:
  • All channels
  • All messages
  • All workspace-user associations
  • All pending invites
✓ Atomic operation (all-or-nothing)
✓ Redirect to workspace selector on success
